# Sample Project

This example package is the quickest way to see what PyDoc2Markdown feels like
on a small but realistic Python project.

It includes:

- typed models, enums, dataclasses, and helper functions
- Google-style docstrings that render into clean Markdown
- a generated docs directory and a generated README API section

Generate the docs for this sample with:

```bash
pydoc2markdown examples/sample_project/src --recursive --nav --readme \
  --readme-path examples/sample_project/README.md \
  -o examples/sample_project/docs
```

That command updates the API section below and writes a browsable docs layout to
`examples/sample_project/docs`.

Inspect the sample first without writing files:

```bash
pydoc2markdown examples/sample_project/src --recursive --doctor
```

Good places to inspect after running it:

- [`src/shop_demo/`](src/shop_demo/) for the input code
- [`docs/index.md`](docs/index.md) for the navigation entrypoint
- [`docs/shop_demo.md`](docs/shop_demo.md) for the package landing page

Try a few filtering scenarios on the same sample:

```bash
# Keep output focused on the exported API surface
pydoc2markdown examples/sample_project/src --recursive --public-only -o examples/sample_project/docs

# Show internal helpers and private state while reviewing internals
pydoc2markdown examples/sample_project/src --recursive --show-private-members -o examples/sample_project/docs

# Keep default output, but hide one noisy member explicitly
pydoc2markdown examples/sample_project/src --recursive --member-exclude "Inventory._validate_product" -o examples/sample_project/docs
```
